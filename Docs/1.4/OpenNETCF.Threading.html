OpenNETCF.Threading Namespace

Namespace hierarchy

Classes

Class Description
EventWaitHandle Represents a thread synchronization event.
Monitor2 Enhanced alternative to the Monitor class. Provides a mechanism that synchronizes access to objects.
Mutex2 A synchronization primitive than can also be used for interprocess synchronization.
Semaphore Limits the number of threads that can access a resource, or a particular type of resource, concurrently.
SemaphoreFullException The exception that is thrown when the Release method is called on a semaphore whose count is already at the maximum.
SynchronizationLockException Exception thrown by Monitor2 when threading rules are violated (usually due to an operation being invoked on a monitor not owned by the current thread).
Thread2 Creates and controls a thread, sets its priority, and gets its status.
WaitHandleCannotBeOpenedException The exception that is thrown when an attempt is made to open a system mutex or semaphore that does not exist.

Enumerations

Enumeration Description
EventResetMode Indicates whether an EventWaitHandle is reset automatically or manually.
ThreadState Specifies the execution states of a Thread2.
